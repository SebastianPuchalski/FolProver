## Description

FolProver is a saturation-based refutational theorem prover for first-order logic with equality.

The repository consists of the prover core, the TPTP tool, and a test suite. The core contains all modules independent of the TPTP format. The TPTP tool is responsible for loading TPTP files (`*.p`, supporting CNF and FOF formulas) and printing proofs in TSTP, UTF-8 text, and HTML formats. The test suite contains several hundred unit tests (mostly LLM-generated).

The system currently implements several solvers and will likely be extended with more advanced (optimized or experimental) ones in the future. Since solvers require Clausal Normal Form (CNF) input, the prover is equipped with a transformation from full First-Order Form to CNF, based on a general formula rewriting mechanism.

The primary solver (`SuperpositionSolver`) implements the ordered superposition calculus and operates within a given-clause algorithm. To strictly constrain the search space, inferences are controlled by a combination of configurable literal selection strategies and ordering restrictions based on the Lexicographic Path Ordering (LPO). Clause selection is managed by a weighted round-robin scheduler that alternates between multiple priority queues, allowing the combination of various heuristic evaluation functions (based on metrics such as clause age and syntactic length). A critical component of the solver is its redundancy elimination mechanism, which performs full forward and backward simplification utilizing an extensive set of contraction rules. Finally, the solver fully supports proof reconstruction and answer extraction.

Note that the TPTP Lexer and Parser modules were primarily generated by LLMs and, despite manual testing and debugging, may contain bugs.

## Installation

The repository contains two independent Visual Studio 2022 projects: the TPTP prover tool (`TptpTool`) and unit tests (`Tests`). The `Tests` project depends on GoogleTest. To use and test the prover, download the TPTP library from http://www.tptp.org. You must set the `TPTP_DIR` system environment variable pointing to the library's root directory.

## Usage

Run the TPTP prover tool from the command line:

```bash
TptpTool.exe [options] <input_file>
```

### Options
* `-t, --time-limit <sec>`: CPU time limit in seconds.
* `-m, --memory-limit <MB>`: Memory limit in megabytes.
* `-p, --proof`: Output proof if found (printed to `stdout` in TSTP format, saved to `lastProof.txt` and `lastProof.html`).
* `-s, --solver <solver_name>`: Choose solver engine. Available: `superposition` (default), `naive-superposition`, `naive-resolution`.
* `-a, --answer-predicate <symbol>`: Set the answer predicate symbol for solution extraction (Green's trick).
* `-h, --help`: Show command-line help.

### Example
```bash
TptpTool.exe -t 60 -p -s superposition Problems/GRP/GRP001-1.p
```
